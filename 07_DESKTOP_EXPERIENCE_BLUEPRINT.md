# Desktop Experience Blueprint

## Desktop feeling

Desktop should feel:
- cinematic
- layered
- editorial
- premium
- immersive
- controlled

It should not feel like a dashboard home screen or a marketing template with boxes stacked in one column.

## Desktop design principle

Use space, contrast, and asymmetrical composition to make the user feel they are stepping into a premium decision environment—not filling out a utility tool.

## `/` Landing page desktop composition

### Block 1 — Hero
**Purpose:** create the immediate “I could actually drive that?” moment.

**Composition:**
- left: strong headline, subhead, CTA stack
- right: large cinematic image or image-plus-proof collage
- floating comparison proof chip layered onto media
- premium badge above headline, restrained and credible

**Emotional job:** aspiration + surprise

### Block 2 — Product truth strip
**Purpose:** clarify what RideRational actually does before confusion sets in.

**Composition:**
- horizontal strip with 3 short statements:
  - local listing hunting
  - sweet-spot economics
  - daily email delivery

**Emotional job:** trust

### Block 3 — Boring default vs sweet spot module
**Purpose:** dramatize the contrast without promising too much.

**Composition:**
- split comparison card
- left panel = default/obvious choice
- right panel = sweet-spot alternative
- central narrative caption
- subtle locked detail cues

**Emotional job:** reframe the category

### Block 4 — Category taste section
**Purpose:** make the product feel personal and desirable.

**Composition:**
- section intro on left
- 2 x 3 category card grid on right
- cards use rich imagery, not plain icon blocks

**Emotional job:** desire + self-recognition

### Block 5 — How it works
**Purpose:** reduce uncertainty.

**Composition:**
- 3-step horizontal flow
- editorial numerals or short labels
- one larger highlighted middle step

**Emotional job:** ease

### Block 6 — Teaser invitation CTA
**Purpose:** move users into the funnel.

**Composition:**
- wide premium panel with single CTA focus
- short supporting line
- light progress-preview motif

**Emotional job:** action

### Block 7 — Trust module
**Purpose:** handle skepticism.

**Composition:**
- narrow text band plus proof tiles
- themes: no marketplace overload, no generic calculator, no full-time browsing required

**Emotional job:** relief

### Block 8 — Trial / pricing signal
**Purpose:** qualify without overwhelming.

**Composition:**
- clean pricing card
- brief 7-day trial language
- understated plan framing

**Emotional job:** commercial clarity

### Block 9 — Final CTA
**Purpose:** close with simplicity.

**Composition:**
- centered final line
- one primary CTA
- one secondary text link

**Emotional job:** commitment

## `/discover` desktop composition

### Layout
- centered shell with generous side breathing room
- top progress indicator
- main step card
- supporting microcopy rail on side or below depending on step

### Step-specific treatment

#### Category step
- 2 x 3 grid of large cards
- each card image-led
- short descriptor under title

#### Purchase mode step
- 3 horizontal selection cards
- one highlighted default state only after user action, not preselected

#### Budget step
- split numeric inputs with contextual helper card
- right-side explainer showing “comfort zone, not max approval”

#### Market step
- tighter form
- Vancouver prefilled
- optional future-scope note minimized

**Emotional job:** fast clarity, not admin burden

## `/teaser` desktop composition

### Block 1 — Personalized headline
**Purpose:** make the result feel tailored without overclaiming.

### Block 2 — Comparison centerpiece
**Purpose:** show benchmark default vs sweet spot alternative.

**Composition:**
- two premium cards
- benchmark card visually calmer
- sweet-spot card visually elevated
- center connector narrative or arrow logic

### Block 3 — Why this is interesting
**Purpose:** connect economics to emotion.

**Composition:**
- concise narrative panel
- one key sentence
- one supporting explanation line

### Block 4 — Local opportunity count
**Purpose:** create urgency without full access.

**Composition:**
- large numeric count
- muted local qualifier
- subtle motion-ready count-up

### Block 5 — Locked preview row
**Purpose:** imply depth of product without cannibalizing paid value.

**Composition:**
- 3 to 4 blurred micro-cards
- lock labels
- “full shortlist starts in your trial” language

### Block 6 — Trial CTA band
**Purpose:** convert.

## `/setup` desktop composition

### Layout
- structured form shell
- left = step explanation and what happens next
- right = editable cards stacked in clear order

### Sections
1. category confirmation
2. spending comfort zone
3. optional refinements
4. email delivery timing
5. next-step summary

**Emotional job:** feel short, intelligent, and manageable

## `/trial` desktop composition

### Layout
- centered commercial shell with side reassurance panel
- pricing card anchored in the visual center
- supporting product-value bullets beside it

**Emotional job:** clarity and trust, not pressure

---

## `/account` desktop composition

### Layout
- no dashboard grid
- account summary first
- settings/action cards second
- expectation-setting strip third

### Key note
The account page confirms that the system is now working on the user’s behalf. It does **not** become a deal-browsing dashboard.

---

## `/settings` desktop composition

### Layout
- clean two-column shell
- sticky section nav optional if density increases
- grouped cards:
  - preferences
  - notifications
  - billing
  - account actions

**Emotional job:** calm maintenance, not product exploration

## Image-led vs UI-led guidance

### Image-led
- landing hero
- category selection
- teaser emotional framing

### UI-led
- setup flow
- settings
- trial state
- account confirmation

## MVP scope note

A deal-browsing dashboard is explicitly not part of the desktop blueprint. Any future browsing surfaces must be separately scoped later.
