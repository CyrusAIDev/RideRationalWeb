# Mobile Experience Blueprint

## Mobile feeling

Mobile should feel:
- immediate
- high-conversion
- thumb-friendly
- visually rich but tighter
- premium without excess
- easy to skim

Desktop may be cinematic and layered. Mobile should be focused and decisive.

## Mobile principles

- shorten section stacks
- reduce copy density
- keep hierarchy obvious within one screen height
- use sticky CTA behavior when it increases conversion clarity
- tighten image crops for emotional punch

## Global mobile rules

- 20px side padding by default
- large thumb-safe tap targets
- short lines of text
- progressive disclosure over giant forms
- a persistent progress bar in flows
- CTA visibility near bottom edge when appropriate
- only one dominant action per viewport

## `/` Landing page mobile composition

### Block 1 — Hero
**Order:**
1. premium badge
2. headline
3. short subhead
4. primary CTA
5. secondary text link
6. tighter hero image

**Emotional job:** surprise quickly

### Block 2 — Product truth chips
**Purpose:** remove ambiguity fast.

### Block 3 — Comparison proof card
**Purpose:** deliver the hook in one swipe-stable module.

### Block 4 — Category carousel/grid
**Purpose:** show the product understands taste.

### Block 5 — How it works
**Purpose:** reduce friction with a simple stacked 3-step flow.

### Block 6 — Trial teaser
**Purpose:** establish the commercial model cleanly.

### Block 7 — Final CTA
**Purpose:** close without noise.

## `/discover` mobile composition

### Layout
- full-width flow shell
- sticky progress bar
- one step per screen
- bottom CTA rail

### Category step
- stacked rich cards or 2-up grid depending on device width
- text kept very short
- selected state very obvious

### Purchase mode step
- large tap cards
- helper note below, not in side rail

### Budget step
- single-column inputs
- contextual guidance collapsible below

### Market step
- simplified single market control
- Vancouver preselected
- minimal friction

**Emotional job:** keep momentum high

---

## `/teaser` mobile composition

### Block order
1. headline
2. short explanation
3. benchmark card
4. sweet-spot card
5. contrast narrative
6. local opportunity count
7. locked preview cards
8. CTA
9. trust note

### Mobile-specific adjustment
On mobile, stack the comparison rather than squeezing a true side-by-side layout. The emotional rhythm matters more than preserving desktop symmetry.

## `/signup` mobile composition

### Layout
- single premium form card
- short context line above
- password helper below field
- CTA fixed inside easy thumb reach

---

## `/setup` mobile composition

### Layout
- step-grouped accordion cards or stacked panels
- only one dense input region visible at a time
- save/continue CTA persistent near bottom when scrolling

### Order
1. categories
2. purchase mode and budgets
3. optional refinements
4. email preferences
5. next-step summary

**Emotional job:** concise and manageable

---

## `/trial` mobile composition

### Layout
- benefit summary first
- pricing card second
- CTA third
- cancellation reassurance last but visible

### Important note
Do not overexplain billing on mobile. State the trial clearly, then keep moving.

---

## `/account` mobile composition

### Layout
- status card first
- what happens next block
- preferences summary
- notifications summary
- billing/manage links

### Emotional job
Make the user feel set up, not stranded.

---

## `/settings` mobile composition

### Layout
- stacked cards
- section labels
- large controls
- destructive actions separated visually and spatially

### Emotional job
Keep edits friction-light.

## CTA persistence strategy

Use sticky bottom CTA bars in:
- discovery flow
- signup
- setup flow
- trial page

Do not use persistent CTAs on:
- long-form settings edits unless actively in a form action
- informational-only sections

## Mobile image crop strategy

- crop tighter on vehicle details and silhouette
- avoid wide empty road scenes on mobile
- keep visual drama in the upper half of the frame
- ensure text overlay never fights the focal point

## MVP scope note

A deal-browsing dashboard is not part of the mobile MVP. Mobile account states should reinforce email delivery and editable preferences, not simulate a search app.
